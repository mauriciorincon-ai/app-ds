Probeta DS — Blueprint de infraestructura as-built

Ciclo: H1 — "El veredicto honesto → sobrevive datos reales" · Cerrado: 2026-07-19 · Sprints del ciclo: S001–S004 (4) · Última actualización: 2026-07-19

La forma de esta infraestructura es el producto. Probeta DS es local-first radical: TODO el cómputo pesado (perfilado, saneamiento, EDA, entrenamiento, scoring, export/import) corre en el navegador del usuario con Pyodide/WASM (pandas + scikit-learn). Los datos del usuario JAMÁS salen del dispositivo (regla dura 2). El servidor es finísimo: Vercel sirve estáticos + una sola ruta serverless (/api/narrate), opt-in y con fallback determinista. No hay base de datos en H1.

Base de datosNINGUNA — local-first (ADR-001)El cómputo corre en el navegador; los datasets viven en memoria/OPFS del cliente. Supabase entra en H2 solo para metadata (proyectos, métricas, model cards) con RLS desde la 1ª tabla — jamás datos crudos del usuario.
AuthNo aplica (H1)Sin cuentas: la app es de un solo uso local. Auth llega con la BD de metadata en H2.
Almacenamiento de archivosEl disco del usuario (descargas del navegador)El modelo (.probeta.json), el CSV puntuado y la model card se descargan al equipo del usuario vía Blob; nada se sube. Sin storage de servidor.
Email / notificacionesNo aplicaLa app no envía correo ni notificaciones.
IA embebidaGroq (OpenAI-compatible) — free tierÚnica IA: la narración del porqué (desde S2, ADR-005/006), opt-in y verificada contra los números. Envía solo agregados (nombres de columna, métricas, alertas EDA) — nunca filas. Adapter conmutable por env; en CI corre el proveedor mock. Guardrails + circuit breaker con fallback determinista.
ObservabilidadSentry + PinoSentry client-side, inerte sin NEXT_PUBLIC_SENTRY_DSN, solo tipos+metadatos (nº filas/columnas), jamás contenido del dataset. Pino registra el costo de cada narración (~US$0.0005) sin datos del usuario.
Crons / keep-alivesNingunoNo hay procesos programados (no hay servidor con estado que mantener vivo).
Secretos.env.local (gitignored) + Vercel env varsDoble cinturón gitleaks: hook PreToolUse de Claude Code + githooks/pre-commit (100755, verificado vivo en S3/S4 con carnada canónica). El repo nunca guarda secretos.
DominioSubdominio de VercelSin dominio propio en H1; se administra desde el panel de Vercel. Dominio propio es decisión de H2 (lanzamiento).
Costo real / mes≈ US$0Vercel Hobby: US$0 (dentro del free tier; app estática liviana). GitHub: US$0 (repo público). Groq: US$0 en la práctica (narración opt-in, ~US$0.0005/uso, medida por Pino; techo declarado US$10/mes). Sentry: US$0 (free tier, inerte sin DSN).
Punto único de fallaVercel (hosting de los estáticos)Si Vercel cae, no se puede cargar la app; pero una vez cargada, corre 100% en el navegador (sin dependencia de servidor para su función central — entrenar/puntuar/exportar). La única llamada de servidor (/api/narrate) es opt-in y degrada a plantilla determinista si Vercel o Groq fallan. Mitigación: estáticos cacheables + degradación honesta; no hay estado central que perder (no hay BD).
